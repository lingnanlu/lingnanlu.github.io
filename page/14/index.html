kramdown syntax - 关键概念

发表于 2015-07-14

一、kramdown 元素

kramdown元素分为 Block-level Elements 和 Span-Level Elements

Block-level Elements 包括

  • Paragraphs
  • Headers
  • Blockquotes
  • Code Blocks
  • Horizontal Rules
  • Lists
  • Definition Lists
  • Tables
  • 其它

Span-Level Elements 包括

  • Emphasis
  • Links and Images
  • Inline Code
  • Footnotes
  • Abbreviations

关于kramdown元素，不必记忆，现用现学，但学的时候要注意以下几点

  1. 元素的表示法

    如Blockquotes的表示是以>开头，后跟一个可选的空格，并且每一行的开头都为>。

    而Code Blocks的表示又有两种，一种是 以4个空格 缩进，另一种不用缩进，但是使用 **

This is a code block

another code block
~~~

关键点：

找到其直接外层元素的第一列的位置(不同Block元素的第一列位置也不同，可参见文档)，然后对齐该位置。

深入理解Yii笔记 - 属性(Property)

发表于 2015-07-10

属性与成员变量

属性表征的是一个对象的特征，如灯这个对象，颜色就是其特征,成员变量表征的是一个对象的结构组成，如灯这个对象，灯泡是其组成。

在面向对象语言中，属性往往由一个成员变量来表示，但这不代表成员变量就是属性

Object中与属性有关的方法

  1. __get(): 调用相应的getName方法返回属性值
  2. __set(): 调用相应的setName方法设置属性值
  3. __isset(): 用于测试属性值是否不为 null ，在 isset($object->property) 时被自动调用。 注意该属性要有相应的getter。
  4. __unset(): 用于将属性值设为 null ，在 unset($object->property) 时被自动调用。 注意该属性要有相应的setter
  5. hasProperty(): 测试某一对象是否有某个属性，只要定义了getter或setter,就认为有这个属性。当传入的参数$checkVars = true时，则只要该对象有同名的成员变量，也认为是具有该属性。
  6. canGetProperty(): 测试某一属性是否可读，只要定义了getter，就认为可读。如果$checkVars = true，只要定义了同名成员变量，就认为可读
  7. canSetProperty(): 测试某一属性是否可写，只要定义了setter，就认为可读。如果$checkVars = true，只要定义了同名成员变量，就认为可写

Object与Component关系

Component相比Object多了 事件 和 行为

同时，Component重载了set()和get()方法，这两个方法的行为类似于Object的行为，只是因为Component多了 行为 ， 行为 会将其中的属性注入到该Component中，所以这两个方法做了稍微的修改。

属性与配置的关系

一个对象存在，必须有某种状态或者说，有某种属性，而在对象的构建过程中，如何获得其所要求的属性呢？这里就需要配置文件，程序启动后，将配置文件的属性读入。所以说，配置文件配置的是对象的属性，如数据库对象的密码和用户名等。

深入理解Yii笔记 - 事件(Event)

发表于 2015-07-10

有关事件的几个关键问题

  1. 事件由谁产生？

    有三种情况：

    • 由某一Component实例产生，这称为实例事件
    • 由某一类产生，这称为类事件
    • 由某一全局可访问的实例产生，如全局唯一的Application实例，这称为全局事件

  2. 什么是事件的产生？
    因为事件发生时，必定有东西来做出响应，即处理事件，这里，事件的产生实际上指的是事件 的trigger。

    • 对于实例事件和全局事件，是由实例的trigger方法触发的
    • 对于类事件，是由Event类的static方法trigger触发的。

  3. 当事件发生(产生，触发，调用trigger)时，如何处理？

    首先，将某些处理者与某一事件进行绑定.

    • 对于实例事件和全局事件，是由实例的on方法绑定的

    • 对于类事件，是由Event类的static方法on绑定的。

      绑定后，事件发生(产生，触发，调用trigger)时,使用用绑定的handler来依次处理该事件。

  4. 事件抽象在程序中如何表示？

    首先有一个事件名，然后还可以有一个Event对象，为该事件提供额外的一些信息和数据。

  5. 事件处理者有哪些？

    • 全局函数
    • 对象方法或类静态方法
    • 匿名函数

  6. 所涉及的数据结构有哪些？

    所谓的绑定就是一个二维数组，第一维是事件名，第二维是绑定到这个事件上的handler所组成的一个数组。

综上所说，首先事件由某一Component实例或Event与handler进行绑定，然后，由某一Component实例或Event进行触发，相应的handler进行处理。

深入理解Yii笔记 - 行为(Behavior)

发表于 2015-07-10

行为是什么

行为就是一种遵循某种规范的类。

行为并不是PHP的语言特性，而可以说是一种方法学，一种设计模式，用来在不修改现有类的情况下为类添加特有的行为(注入属性，方法之类)

行为的绑定

  1. 行为只能绑定到Component
  2. 绑定是双向的，需要Component和Behavior同时做工作。
  3. 所谓的绑定也是Components中维护一个数组，其元素是所绑定的行为，同时，行为类维护一个成员$owner，指向其所绑定的Component。
  4. 行为也可以对事件做出响应，即处理事件，当绑定某一行为到Component时，会将行为所绑定的事件注册到其依附的Component中去。

行为的一个应用

yii\behaviors\TimeStampBehavior是yii内置的一种行为，其绑定了两个事件，
BaseActiveRecord::EVENT_BEFORE_INSERT
BaseActiveRecord::EVENT_BEFORE_UPDATE
当这两个事件发生时，会更新相应字段的时间戳。

此时，我们可以定义一个Component做为Model，将TimeStampBehavior绑定到该对象上，当往数据库插入或更新一条记录时，会发生相应的事件，然后自动更新相应的字段。
