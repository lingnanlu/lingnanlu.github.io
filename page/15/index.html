什么是代码高亮？

代码高亮就是为代码添加相应的html tag，并为不同的tag添加不同的样式。

如何实现代码高亮

可以使用3种方法

  1. 使用Markdown解析器

    此时代码的格式要符合Markdown所要求的格式。Markdown解析器会为代码添加相应的html tag，并调用相应插件来为代码添加样式。

    如：使用kramdown和coderay为代码添加样式。

    注：这种方式github pages并不一定支持，因为kramdown需要调用coderay来为代码添加样式，而github pages不一定有coderay

  2. 使用liquid模板语言

    此时代码的格式要符合liquid模板语言所要求的格式。liquid模板语言为代码添加相应的html tag和样式。

    github pages支持liquid模板语言调用Pygments来实现。

  3. 使用js

    使用Markdown解析器为代码添加html tag，但并不添加样式，在网页载入过程中，调用相应的代码高亮js库来实现。

    如: kramdown + google-code-prettify

encoding

默认情况下，print str是向标准终端文件写字符串str，既然写字符串，那么就需要指定所写内容的编码。

实际上str对象有一个encoding("codecs")方法，来显示的指定所使用的encoding.

但是当不显示指定时，所使用的encoding是什么呢？

默认的encoding是由sys.stdout.encoding来指定的，而sys.stdout.encoding是由什么进行初始化的呢？

在shell环境下，sys.stdout.encoding是由shell的环境变量所初始化的

decoding

以上是encoding，那么print str后，是在终端显示的，这里，在终端显示是类似与读文件的过程，所以所用终端需要设置decoding，当encoding与decoding一致时，就可以正常显示了。

关键点

  1. 写文件的过程
  2. 读文件的过程

静态类型语言
: 一种在编译期间就确定数据类型的语言。大多数静态类型语言是通过要求在使用任一变量之前声明其数据类型来保证这一点的。Java 和 C 是静态类型语言。

动态类型语言
: 一种在运行期间才去确定数据类型的语言，与静态类型相反。VBScript 和 Python 是动态类型的，因为它们确定一个变量的类型是在您第一次给它赋值的时候。

强类型语言
: 一种总是强制类型定义的语言。Java 和 Python 是强制类型定义的。您有一个整数，如果不明确地进行转换 ，不能将把它当成一个字符串。

弱类型语言
: 一种类型可以被忽略的语言，与强类型相反。VBScript 是弱类型的。在 VBScript 中，您可以将字符串 ‘12’ 和整数 3 进行连接得到字符串’123’，然后可以把它看成整数 123，所有这些都不需要任何的显示转换。

所以说 Python 既是动态类型语言(因为它不使用显示数据类型声明)，又是强类型语言(因为只要一个变量获得了一个数据类型，它实际上就一直是这个类型了)。

第13章 单元测试

编写测试用例的原则

  1. 每个测试用例只回答一个问题，试想，如果试着回答多个问题，如果测试失败了，还要分析到底是一个部分出了问题
  2. 每个测试用例必须与其它测试用例隔离工作，其它用例通不通得过与本测试用例应该无关。

今天的单元测试就是明天的回归测试，使用单元测试可以保证在修改代码后，不影响代码原来的功能。

单元测试的基本理念:

  1. 测试用例的设计方针是目的单一、可以自动运行、互不干扰
  2. 在被测试代码编写之前编写测试用例
  3. 编写测试有效输入的测试用例并检查正确的结果
  4. 编写测试无效输入的测试用例并检查正确的失败
  5. 为描述Bug或反映新需求而编写和升级测试用例
  6. 为改进性能、可伸缩性、可读性、可维护性和任何缺少的特征而无情地重构。

闭包的一个例子

发表于 2015-03-31

定义

闭包是一个函数，在这个函数体中引用了某一变量，而该变量的定义是在该函数外。闭包函数和这个变量将一同存在，即使该变量已经离开了创造它的环境。

例子

def scaledata(rows):
  low=[999999999.0]*(len(rows[0].data) - 1)
  high=[-999999999.0]*(len(rows[0].data) - 1)
  # Find the lowest and highest values
  for row in rows:
    d=row.data
    for i in range(len(d) - 1):
      if d[i]<low[i]: low[i]=d[i]
      if d[i]>high[i]: high[i]=d[i]

  # Create a function that scales data
  def scaleinput(d):
     return [(d[i]-low[i])/(high[i]-low[i])
            for i in range(len(low))] + [d[len(d) - 1]]

  # Scale all the data
  newrows=[matchrow(scaleinput(row.data)+[row.match])
           for row in rows]

  # Return the new data and the function

  return newrows,scaleinput

这里scaleinput是一个闭包，它保存了变量low, high当scaledata返回时，low和high依然存在。这样，当对新数据进行缩放时，不必再计算low和high值。

如果不使用闭包，有什么替代方法呢？

方法1：如果scaleinput定义在scaledata之外，则需要保存high和low，并将它们作为参数传递给scaleinput

方法2：同方法1，将high与low定义为全局的。

方法1和方法2的耦合度太高，利用闭包降低了耦合度，scaledata实际上作的是返回了一个数据集rows上的属性(high, low)和操作(scaleinput)，从这个角度来看，类也能实现类似功能。

搜索的大体流程

  1. 利用爬虫抓取数据，如网页
  2. 将这么数据保存到数据库中并建立索引。
  3. 查询数据库得到结果
  4. 利用一定的算法对查询结果进行排序

    • 文档位置
    • 单词距离

神经网络

为什么需要一种像神经网络这样的复杂技术，而不是简单地记录下查询地记录下查询条件以及每个搜索结果被点击的次数呢？

这是因为神经网络可以根据与其他查询的相似度情况，对以前从未见过的查询结果做出合理的猜测。

比如说，输入法，当你输出某个首字母缩写后，从候选词列表中选择一下，当选择次数多之后，再次输入时，该被选择词组会被放在第一个位置。

当你输入另一个首字母缩写（该单词的含义与之前类似，如计算机和电脑），神经网络可以利用之前的训练给出合理的结果，如果不使用神经网络，仅仅是简单的记录，是无法利用之前的查询结果的。

为什么神经网络可以做到呢？这是因为，之前的查询会对某些神经元进行训练，这些训练会不断加强相关神经元的活跃程度，之后，再遇到类似的查询时这些活跃的神经元就可以帮助给出合理的结果。

关键问题

神经网络一个关键问题是，如何利用用户对搜索结果的偏好来改变各神经元之间的链接强度。

其它概念

归一化处理：对结果进行处理即令之有相同的值域和变化方向。

第五章 优化

本章要解决的问题

对于一个问题可能有很多解，这些解有的好有的不好，而我们不能对所有可能的解进行一一尝试来找出最佳解，优化算法就是从一种解出发，然后对其进行不断的修正，从而使其接近最优解。

关键是确定

  1. 解的表示方法
  2. 一个成本函数用来评估解的优劣

典型算法

  1. 随机搜索：在解域随机选择若干，找出其中的使成本函数值最小的解。该方法不能找出最优解。

  2. 爬山法：先随机找出一个解，然后找出其所有邻居解。在邻居解中找出最优解，再迭代重复。

    形象化：解为横坐标，解的成本为纵坐标。

    优缺点：因为初始解是随机的，所以只能得到局部最优解。做为改善，可以使用随机重复爬山法。

  3. 模拟退火：该算法也是先找一个随机解，与爬山法不同的是，该算法有一定的概率接受一个更差的解，而且该概率与一个变量——温度有关，当温度高时，接受更差的解的可能性比较大，随着迭代的进行，温度慢慢下降，接受更差的解的可能性也逐渐降低。

  4. 遗传算法：随机选择一组解，然后从这组解中选取精英解，利用这些精英解，构建一组新解。构建方法有

    • 变异
    • 交叉

优化算法起作用的前提是：最优解应该接近于其他的优解。

应用实例

  1. 航班安排
  2. 学生宿舍优化(资源分配问题)
  3. 绘制一个交叉线尽量少的网络

第八章 构建价格模型

解决问题

本章要解决的问题是：利用一个数据集，对数值型数据进行预测。

使用的算法

  1. K-最近邻算法：利用与待预测的商品相近的商品价格对该商品的价格进行预测。

应用

  1. 商品价格预测

算法流程(kNN)

  1. 选择一组对待预测数据产生影响的变量
  2. 构造数据集，利用此数据集对待预测数据进行预测

关键问题(kNN)

  1. 选择相近的商品数(k值的决定)
  2. 对不同距离的邻居分配不同权重(一般越远权重越小)
  3. 对预测算法的准确性进行验证(利用交差验证函数)
  4. 一组变量中，不同变量的值域并不相同，直接计算距离值是没有意义的。而且还可能引入了完全不相关的变量。为了提高预测的准确度，需要对各变量进行 按比例缩放 处理。至于对各变量的比例
    缩放大小是多少，可以利用优化算法来进行确定(使用优化算法时，成本函数可以是交差验证函数)

优点与不足(kNN)

优点：因为每一次预测都要计算近邻，所以将新的观测数据加入到数据集中时，无须任何计算开销(其它算法可能在数据集发生改变时重新进行训练)

缺点：

  1. 计算量很大

第九章 核方法与SVM

名词

线性分类的工作原理：寻找每个分类中所有数据的平均值，并构造一个代表该分类中心位置的点。然后判断待分类数据距离哪个中心点近来确定其分类。线性分类器需要找出一条分界线来，根据
点与分界线的相对位置来判断属于哪一类

核函数：当将一个空间A变换到另一个空间B（利用映射）,对于空间B，计算空间B中的两个向量的内积，并使用空间A中的变量表达该内积，这样的一个函数表达式就是核函数

径向基函数：是这样的一种函数，其值只依赖于变量距某一点的距离。

支持向量机：是一种分类器，利用支持向量来寻找分界线。支持向量是每一类上的边缘处那几个点分界线的位置是完全由支持向量来确定，也就是说，如果去掉每一类的非支持向量点，分界线的位置是不变的。在某些情况下，在低维度，寻找一个分界超平面是一个非线性的问题，此时，需要把数据空间从低维度映射到高维度，映射到高维度之后，就可以找到一个分界超平面来进行分类。

注意：核函数和径向基函数没有必然联系，只是有的核函数的形式就是一种径向基函数。

第十章 寻找独立特征

关键问题

本章要解决的问题是，从一组数据集出，提取出模式，即，一些数据属于某一模式，另一些属于另一模式，本章就是从大量数据集中找出隐含在其背后的模式。

应用实例

  1. 将大量新闻进行分组。找出新闻背后的主题
  2. 股票市场

使用算法

NMF：非负矩阵因式分解。

将一个矩阵分解为两个矩阵，一个为特征矩阵， 一个为权重矩阵。这两个矩阵中的元素都是非负的。特征矩阵和行数以及权重矩阵的列数就是要找的特征数(模式数)。所以寻找背后的模式也称为特征提取。

第十一章 智能进化

关键问题

本章利用遗传的思想来构造程序。

解决实际问题时，总是要寻找一个解，即构造出一个程序来。但有的时候，凭借人工无法找出这样的一个程序，如随机给定一些点，寻找一个函数来拟合这些点。

遗传编程的思想时，先随机生成很多个程序来，然后对这些程序选出优异的个体，进行交叉，变异传递给下一代，这样进化几代之后，就可能找出这样一个程序，可以解决要解决的问题。

由上可知，在使用遗传编程时，有如下两点需要注意。

  1. 寻找一种程序的表示方法，从而方便的对程序进行变异，交叉等(使用树的表示法)
  2. 对程序进行评价，衡量程序的好坏。

应用实例

  1. 寻找未知函数
  2. 在游戏中，构造智能玩家。
