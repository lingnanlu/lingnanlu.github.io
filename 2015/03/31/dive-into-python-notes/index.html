静态类型语言
: 一种在编译期间就确定数据类型的语言。大多数静态类型语言是通过要求在使用任一变量之前声明其数据类型来保证这一点的。Java 和 C 是静态类型语言。

动态类型语言
: 一种在运行期间才去确定数据类型的语言，与静态类型相反。VBScript 和 Python 是动态类型的，因为它们确定一个变量的类型是在您第一次给它赋值的时候。

强类型语言
: 一种总是强制类型定义的语言。Java 和 Python 是强制类型定义的。您有一个整数，如果不明确地进行转换 ，不能将把它当成一个字符串。

弱类型语言
: 一种类型可以被忽略的语言，与强类型相反。VBScript 是弱类型的。在 VBScript 中，您可以将字符串 ‘12’ 和整数 3 进行连接得到字符串’123’，然后可以把它看成整数 123，所有这些都不需要任何的显示转换。

所以说 Python 既是动态类型语言(因为它不使用显示数据类型声明)，又是强类型语言(因为只要一个变量获得了一个数据类型，它实际上就一直是这个类型了)。

续行符\

当一条命令使用续行符分割成多行时，后续的行可以以任何方式缩进。
严格地讲，在小括号，方括号或大括号中的表达式 (如定义一个 dictionary) 可以用或者不用续行符\分割成多行。

sys.path是如何被初始化的？

sys.path按先后顺序包括以下三个方面的内容。

  1. site模块利用.pth文件添加的内容。安装的第三方库往往利用这种方式。

模块内置属性__name__

一个模块的__name__属性的值取决于你如何应用模块。

  1. 如果import模块，__name__的值通常为模块的文件名，不带路径或者扩展名。
  2. 如果直接运行模块， __name__的值将是一个特别的缺省值，__main__。

可以利用此特性来测试模块。

第12章 SOAP Web服务

SOAP和WSDL

SOAP
: Simple Object Access Protocol，提供了”请求“的规范，向服务接口传递请求的格式，包括方法，参数之类的，这种格式使用XML来描述。一个常用的应用是RPC

WSDL
: 是对WebService所提供的服务的一种文档化描述，这种描述使用XML语言。

SOAP和WSDL的关系：有点类似与XML与XSD的关系，WSDL可以验证发送的一个SOAP请求是否合法(比如，调用的方法存不存在，参数正不正确之类的)。

第13章 单元测试

编写测试用例的原则

  1. 每个测试用例只回答一个问题，试想，如果试着回答多个问题，如果测试失败了，还要分析到底是一个部分出了问题
  2. 每个测试用例必须与其它测试用例隔离工作，其它用例通不通得过与本测试用例应该无关。

今天的单元测试就是明天的回归测试，使用单元测试可以保证在修改代码后，不影响代码原来的功能。

单元测试的基本理念:

  1. 测试用例的设计方针是目的单一、可以自动运行、互不干扰
  2. 在被测试代码编写之前编写测试用例
  3. 编写测试有效输入的测试用例并检查正确的结果
  4. 编写测试无效输入的测试用例并检查正确的失败
  5. 为描述Bug或反映新需求而编写和升级测试用例
  6. 为改进性能、可伸缩性、可读性、可维护性和任何缺少的特征而无情地重构。

第15章 重构

全面的单元测试带来的好处

  1. 全部测试用例最终通过时的成就感
  2. 被责怪破坏了别人的代码时能够证明 自己的自信
  3. 给了你自由去无情地重构

生成器

在函数定义中，使用yield返回，此函数便是一个生成器。
每次调用生成器的next()方法，函数便执行到yield语句，并返回。
再次调用next()方法，将从上次yield的地方继续执行。
